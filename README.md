# 简单的音乐游戏原型

## 简介

这是一个个人练手用Unity小项目，旨在尽量做完一个音乐游戏至少需要的界面。为了方便留存，查看和分享上传到Github。

最开始的时候是想要做一点新的玩法的，但是想玩法实在太难，把想法实现出来也很难，实现出来的想法能好玩还是很难。本来的计划最后都被否掉了，实在是没有好的想法，就先去做了其他部分。目前的玩法就是普通无轨下落（可以上落），有大小键有垂直判定。

## 使用方法

在StreamingAssets下创建chart_database.txt，内容为每行一个序号+谱面路径（逗号分隔），同路径下创建Charts文件夹，Charts文件夹中放置乐曲文件夹，一个乐曲文件夹由chart.json（SongChartData类）和music.mp3组成，在游戏中输入序号加载对应谱面

## 还没有做的功能

由于我个人目前的能力，思路，或者其他一些原因的限制，目前有以下功能没有实现：

- 可视化制谱器
- 选关页面
- 暂停/重开
- 分数保存

## 已经实现的功能

- 玩家信息本地保存（姓名，UID，设置项）
- 流速设置，延迟设置，姓名修改
- 向HTTP服务器上传/下载玩家信息（Json文件）（需要服务器端支持）
- 读取谱面列表，根据输入的序号加载存放在对应路径的谱面（.json）和音乐（.mp3）
- 根据谱面信息完成音符的生成，显示，输入和判定逻辑。
- 游玩中的分数连击判定显示UI，游玩结束后的结算UI
- 对UI进行了一定程度的布局，尽量在不同设备上都能正常显示
- 对16:9和4:3的屏幕比例进行了游玩区域的适配，尽量减小不同设备游玩的差异

2. 由于没有制谱器，目前写谱方法为各种格式转Json

5. 由于整体耗时也有不短的一段时间（虽然很多时间都是在左右互博，很多东西加了又删），一开始也没确定变量命名的标准，写的过程中存在帕斯卡，驼峰，下划线，英文缩写混用的情况。
